# Zero-Knowledge Private Vesting Smart Contract

## Zero-Knowledge usage

The contract verifies SNARK proofs that:
- A recipient is included in the vesting Merkle tree
- The claimed amount is valid
- The withdrawal respects vesting rules
- No information about other recipients is revealed on-chain

## Protocol flow

1. Creator builds vesting dataset off-chain:
   - recipients
   - allocations
   - cliff schedule
   
2. Creator generates a Merkle tree from this dataset

3. Creator deploys the contract with:
   - token address
   - Merkle root
   - ZK proof verifying tree correctness
   
4. Users receive individual ZK proofs off-chain

5. Users call `withdraw()` with:
   - ZK proof
   - tokens receiver
   
6. Contract verifies proof and releases tokens

## Privacy guarantees

Public:
- Sum of total allocations
- ERC20 token address
- Merkle root
- Vesting start time
- Already withdrawn amounts

Hidden:
- recipient addresses (before first interaction)
- individual allocations
- cliff schedules
- full vesting table
